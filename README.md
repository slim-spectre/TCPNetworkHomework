### Тлумачення станів TCP з'єднань

> 

1.  **LISTEN** — Порт відкритий програмою і очікує на вхідний запит (режим сервера).
2.  **ESTABLISHED** — Канал зв'язку встановлено, відбувається активний обмін даними.
3.  **TIME_WAIT** — З'єднання завершене, але ОС чекає певний час, щоб прийняти пакети, які могли затриматися в мережі.
4.  **CLOSE_WAIT** — Сервер ініціював закриття, локальна машина готується до завершення сесії.

## Робота з утилітою Telnet
Команда `telnet [host] [port]` використовується для **перевірки доступності порту** на віддаленому сервері. Вона дозволяє встановити пряме TCP-з'єднання та вручну надіслати запит (наприклад, HTTP GET), щоб перевірити, чи відповідає сервер.

---
1. **Діапазони портів:**
   * **0-1023 (Well-known):** Стандартні системні сервіси.
   * **1024-49151 (Registered):** Зареєстровані порти для ПЗ.
   * **49152-65535 (Dynamic):** Тимчасові порти для клієнтських з'єднань.
2. **HTTP (80) vs HTTPS (443):** Розподіл на різні порти дозволяє серверу миттєво визначити тип трафіку — відкритий або зашифрований (SSL/TLS).
3. **Ephemeral ports:** Тимчасові порти, які ОС виділяє програмі (наприклад, браузеру) для ініціації зв'язку. Вони звільняються одразу після закриття сесії.
4. **Два сервери на одному порту:** Одночасно запустити неможливо. Порт може бути зайнятий лише одним процесом (помилка "Address already in use").



## Завдання 3: Експерименти з TCP з'єднаннями
---

### Відповіді на контрольні питання

#### 1. Що відбувається під час встановлення TCP з'єднання (3-way handshake)?
Встановлення з'єднання відбувається у три етапи («трьохетапне рукостискання»):
* **SYN (Synchronize):** Клієнт надсилає запит на встановлення зв'язку з початковим номером послідовності.
* **SYN-ACK (Synchronize-Acknowledgment):** Сервер підтверджує отримання запиту та надсилає свій запит на з'єднання у відповідь.
* **ACK (Acknowledgment):** Клієнт підтверджує отримання відповіді від сервера. Після цього встановлюється стан **ESTABLISHED**.



#### 2. Як сервер розуміє, що клієнт хоче з'єднатися?
Сервер працює в режимі «прослуховування» (**LISTEN**) на певному порту. Коли на цей порт надходить пакет із встановленим прапорцем **SYN**, операційна система розпізнає це як спробу ініціювати нове TCP-з'єднання і передає запит відповідному додатку.

#### 3. Що станеться, якщо порт закритий або фільтрується Firewall?
* **Якщо порт закритий:** Сервер надішле у відповідь пакет із прапорцем **RST** (Reset). Клієнт миттєво отримає помилку *"Connection refused"*.
* **Якщо порт фільтрується Firewall:** Брандмауер просто ігнорує (скидає) пакети клієнта, не надсилаючи відповіді. Клієнт буде очікувати на відповідь, поки не вичерпається ліміт часу, і видасть помилку *"Connection timed out"*.
